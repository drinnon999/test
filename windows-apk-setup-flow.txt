Windows 해커톤 세팅 + 웹서버 + APK 파이프라인
============================================

전제
----
- 대상 환경: 초기화된 Windows 노트북
- WSL은 필요 없음
- 목표: 바이브 코딩은 바로 시작하고, APK 빌드/폰 미러링 파이프라인은 초반에 미리 뚫어둔다
- 기본 전략:
  - 개발은 Git + Node.js + Claude Code + Codex로 진행
  - 웹앱은 Next.js 개발 서버로 실행
  - APK는 Capacitor로 Android WebView 래퍼를 만든다
  - 실제 APK 빌드는 Gradle Wrapper가 한다
  - 시스템 Gradle은 따로 설치하지 않는다
  - 폰에서는 adb reverse로 PC의 http://localhost:3000 개발 서버를 본다

발표 슬라이드 캡처까지 쓸 경우:

npx playwright install chromium


3. Android SDK 명령줄 도구 설치
-------------------------------
JDK 21은 위 winget 명령으로 설치했다. Android SDK는 별도로 구성해야 한다.

$sdk='C:\Android\sdk'

New-Item -ItemType Directory -Force "$sdk\cmdline-tools" | Out-Null
Invoke-WebRequest 'https://dl.google.com/android/repository/commandlinetools-win-11076708_latest.zip' -OutFile "$env:TEMP\clt.zip"
Expand-Archive "$env:TEMP\clt.zip" "$sdk\cmdline-tools\_t" -Force
Move-Item "$sdk\cmdline-tools\_t\cmdline-tools" "$sdk\cmdline-tools\latest"

$env:JAVA_HOME='C:\Program Files\Microsoft\jdk-21.0.11.10-hotspot'
& "$sdk\cmdline-tools\latest\bin\sdkmanager.bat" --sdk_root="$sdk" "platform-tools" "platforms;android-36" "build-tools;36.0.0"

adb가 설치됐는지 확인한다.

C:\Android\sdk\platform-tools\adb.exe version

메모:
- Android Studio는 필요 없다.
- 시스템 Gradle도 필요 없다.
- 나중에 Capacitor가 Android 프로젝트를 만들면 Gradle Wrapper가 같이 생긴다.
- 첫 빌드 때 Gradle Wrapper가 필요한 Gradle 배포판을 인터넷에서 자동 다운로드한다.

브라우저에서 확인한다.

http://localhost:3000

프로덕션 빌드 확인:

cd C:\hackathon\project
npm --prefix web run build


5. APK 파이프라인을 초반에 뚫기
-------------------------------
"APK 파이프라인을 뚫는다"는 뜻:
- 실제 앱이 완성되기 전에 빈 WebView APK라도 한 번 빌드/설치/실행한다.
- JDK, Android SDK, Capacitor, Gradle Wrapper, adb, USB 디버깅, adb reverse, scrcpy가 모두 되는지 확인한다.
- 후반에는 웹 코드만 바꿔도 폰 앱이 PC 개발 서버를 보기 때문에 APK 재빌드가 대부분 필요 없다.

PowerShell 2번 창에서 실행한다.

cd C:\hackathon\project\web
npm install -D @capacitor/core @capacitor/cli @capacitor/android
npm install @capacitor/geolocation

Capacitor가 요구하는 자리표시 webDir을 만든다.

New-Item -ItemType Directory -Force capacitor-www | Out-Null
'<html><body>loading</body></html>' | Set-Content -Encoding UTF8 capacitor-www\index.html

아래 파일을 만든다.

C:\hackathon\project\web\capacitor.config.ts

파일 내용:

import type { CapacitorConfig } from '@capacitor/cli';

const config: CapacitorConfig = {
  appId: 'kr.hackathon.app',
  appName: '해커톤앱',
  webDir: 'capacitor-www',
  server: {
    url: 'http://localhost:3000',
    cleartext: true,
  },
};

export default config;

Android 프로젝트를 생성한다.

npx cap add android
npx cap sync android

이 시점에 아래 파일들이 생긴다.

web\android\gradlew.bat
web\android\gradlew
web\android\gradle\wrapper\gradle-wrapper.jar
web\android\gradle\wrapper\gradle-wrapper.properties
web\android\app\build.gradle


6. 첫 APK 빌드
--------------
중요:
- 첫 Gradle 빌드는 사람이 연 일반 PowerShell에서 실행하는 것을 권장한다.
- 일부 AI 에이전트 터미널은 Gradle의 루프백/소켓 동작을 막을 수 있다.

PowerShell 2번 창:

$env:JAVA_HOME='C:\Program Files\Microsoft\jdk-21.0.11.10-hotspot'
$env:ANDROID_HOME='C:\Android\sdk'

cd C:\hackathon\project\web\android
.\gradlew.bat assembleDebug

생성될 APK:

C:\hackathon\project\web\android\app\build\outputs\apk\debug\app-debug.apk

왜 Gradle을 따로 설치하지 않는가:
- Android 프로젝트는 Gradle Wrapper 방식을 쓴다.
- 우리는 시스템 gradle 명령이 아니라 .\gradlew.bat를 실행한다.
- Gradle Wrapper가 gradle-wrapper.properties에 적힌 버전을 확인한다.
- 해당 Gradle이 PC에 없으면 첫 빌드 때 자동 다운로드한다.
- 대회장에 인터넷이 있다면 이 방식으로 충분하다.


7. 폰 연결, APK 설치, 화면 미러링
---------------------------------
PowerShell 3번 창에서 실행한다.

C:\Android\sdk\platform-tools\adb.exe devices

폰에서 해야 할 일:
- 개발자 옵션 켜기
- USB 디버깅 켜기
- "USB 디버깅을 허용하시겠습니까?" 팝업 허용

그다음 실행한다.

C:\Android\sdk\platform-tools\adb.exe reverse tcp:3000 tcp:3000
C:\Android\sdk\platform-tools\adb.exe install -r C:\hackathon\project\web\android\app\build\outputs\apk\debug\app-debug.apk
C:\Android\sdk\platform-tools\adb.exe shell am start -n kr.hackathon.app/.MainActivity
scrcpy

이 구조의 의미:
- adb reverse가 폰의 localhost:3000을 PC의 localhost:3000으로 연결한다.
- APK는 Android WebView에서 http://localhost:3000을 연다.
- 따라서 폰 앱은 PC의 Next.js 개발 서버 화면을 보고 있다.
- scrcpy는 폰 화면을 노트북/빔프로젝터에 띄운다.


8. 대회 중 권장 창 구성
-----------------------
PowerShell 창을 3개로 나눠 쓴다.

PowerShell 1: 웹 개발 서버

cd C:\hackathon\project\web
npm run dev

PowerShell 2: Android 빌드 작업

cd C:\hackathon\project\web
npx cap sync android
cd android
.\gradlew.bat assembleDebug

PowerShell 3: 폰 연결/시연

C:\Android\sdk\platform-tools\adb.exe devices
C:\Android\sdk\platform-tools\adb.exe reverse tcp:3000 tcp:3000
scrcpy

대부분의 웹 UI 수정은 웹 개발 서버만 켜져 있으면 된다.
APK는 보통 다시 빌드하지 않아도 된다.


9. APK 재빌드가 필요한 경우
---------------------------
아래처럼 Android 네이티브 쪽 설정이 바뀌면 재빌드가 필요하다.

- appId 변경
- appName 변경
- 아이콘/스플래시 변경
- AndroidManifest 권한 변경
- Capacitor 플러그인 추가/삭제
- capacitor.config.ts의 server.url 변경

그때는 아래를 실행한다.

cd C:\hackathon\project\web
npx cap sync android
cd android
.\gradlew.bat assembleDebug
C:\Android\sdk\platform-tools\adb.exe install -r app\build\outputs\apk\debug\app-debug.apk


10. APK 재빌드가 보통 필요 없는 경우
------------------------------------
아래 변경은 대부분 재빌드가 필요 없다.

- React 컴포넌트 수정
- CSS/Tailwind 수정
- Next.js 페이지 수정
- 개발 서버의 API route 수정
- 문구/레이아웃 수정

이유:
- APK가 앱 안에 완성된 웹 파일을 들고 있는 것이 아니다.
- APK가 http://localhost:3000 개발 서버를 WebView로 열고 있다.
- adb reverse가 폰 localhost:3000을 PC localhost:3000으로 터널링한다.


11. 자주 막히는 지점
--------------------
winget 명령이 없음:
- Microsoft Store에서 "App Installer"를 설치/업데이트한다.

node/npm이 설치 후에도 안 잡힘:
- 새 PowerShell을 연다.

adb devices가 unauthorized로 표시됨:
- 폰 화면에서 USB 디버깅 허용을 누른다.
- USB 케이블을 다시 연결한다.

폰 앱에서 localhost 화면이 안 뜸:
- PC에서 npm run dev가 켜져 있는지 확인한다.
- adb reverse를 다시 실행한다.
  C:\Android\sdk\platform-tools\adb.exe reverse tcp:3000 tcp:3000

Gradle 빌드가 JAVA_HOME 때문에 실패:
- 실제 JDK 설치 경로를 확인한다.
  C:\Program Files\Microsoft\
- JAVA_HOME을 그 경로에 맞게 바꾼다.

첫 Gradle 빌드가 느림:
- 정상이다.
- Gradle과 Android 의존성을 처음 한 번 다운로드한다.
- 그래서 APK 파이프라인은 초반에 미리 뚫어두는 것이 좋다.

scrcpy가 실행되지 않음:
- adb devices에서 기기가 잡히는지 먼저 확인한다.
- 필요하면 아래처럼 scrcpy가 사용할 adb를 지정한다.
  $env:ADB='C:\Android\sdk\platform-tools\adb.exe'
  scrcpy
